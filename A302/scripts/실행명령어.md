`A302` 프로젝트 루트에서 실행:

```powershell
cd .\A302
```

엔진 경로 지정 방법(둘 중 하나):

```powershell
# 1) 환경변수로 1회 설정
$env:UE_ENGINE_ROOT="..\..\UnrealEngine"

# 2) 실행 시 파라미터로 직접 전달
powershell -ExecutionPolicy Bypass -File .\scripts\start-local-test-stack.ps1 -EngineRoot "..\..\UnrealEngine"
```

자동 탐색도 지원:
- `C:\UnrealEngine`, `D:\UnrealEngine`
- `C:\Program Files\Epic Games\UE_*` 계열

Force 빌드 후 로컬 스택 실행(로비 + Dedicated):

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start-local-test-stack.ps1 -ForceRebuildStaged
```

Force 빌드 없이 실행(기존 Staged 서버 재사용, 없으면 자동 빌드):

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start-local-test-stack.ps1
```

Dedicated 서버만 실행(로비 서버는 이미 따로 켠 상태):

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start-local-test-stack.ps1 -SkipLobby
```

로컬 테스트 프로세스 종료:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\stop-local-test-stack.ps1
```

사설 서버(Server.exe) 빌드 후 실행:

```powershell
# 빌드
cmake -S ..\Server -B ..\Server\build
cmake --build ..\Server\build --config Release

# 실행
..\Server\build\Server.exe
```

참고:
- `start-local-test-stack.ps1`는 로비 서버 실행 파일을 상대경로로 탐색합니다.
  - `..\Server\build\Server.exe` (우선)
  - `..\Server\GameServer\build\Release\Server.exe`
  - `..\Server\GameServer\build\Server.exe`
- Dedicated 기본 맵은 `-Map` 미지정 시 `/Game/PersonalWorkSpace/wjtmd28/Loading_Level` 입니다.
- 실제 룸 인게임 템플릿 맵은 서버 런타임에서 `/Game/PersonalWorkSpace/wjtmd28/MyMap`을 스트리밍합니다.

## Linux Dedicated 빌드 + EC2 배포

전제:
- Windows에서 Linux Server 크로스 빌드 환경(UE Linux Toolchain)이 구성되어 있어야 합니다.
- EC2(Ubuntu)에 `systemd`, `tar`, `sudo`가 준비되어 있어야 합니다.

Linux Dedicated 빌드 + tarball 생성:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build-linux-dedicated.ps1 -EngineRoot "..\..\UnrealEngine"
```

DDC 캐시 경로 지정(권장):

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build-linux-dedicated.ps1 `
  -EngineRoot "..\..\UnrealEngine" `
  -DdcCachePath "..\Artifacts\UE_DDC"
```

빌드 산출물:
- 디렉터리: `..\Artifacts\LinuxServer`
- tarball: `..\Artifacts\A302_LinuxServer_yyyyMMdd_HHmmss.tar.gz`

EC2 배포(최신 tarball 자동 선택):

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\deploy-linux-dedicated-to-ec2.ps1 `
  -RemoteHost "<EC2_PUBLIC_IP_OR_DOMAIN>" `
  -User "ubuntu" `
  -SshKeyPath "C:\path\to\ec2.pem" `
  -ServiceName "a302-ds-v2.service"
```

원격 배포 구조:
- 릴리즈: `/home/ubuntu/DedicatedServer_A302/releases/<timestamp>`
- 현재 링크: `/home/ubuntu/DedicatedServer_A302/current`
- 실행 호환 링크: `/home/ubuntu/DedicatedServer_A302/LinuxServer` (매 배포 시 최신 릴리즈로 갱신)

systemd 템플릿:
- `.\scripts\a302-dedicated.service.example`를 EC2에 복사해 사용
- 등록 예시:

```bash
sudo cp a302-dedicated.service /etc/systemd/system/a302-ds-v2.service
sudo systemctl daemon-reload
sudo systemctl enable a302-ds-v2.service
sudo systemctl start a302-ds-v2.service
```
