`A302` 프로젝트 루트에서 실행:

```powershell
cd .\A302
```

엔진 경로 지정 방법(둘 중 하나):

```powershell
# 1) 환경변수로 1회 설정
$env:UE_ENGINE_ROOT="..\..\UnrealEngine"

# 2) 실행 시 파라미터로 직접 전달
powershell -ExecutionPolicy Bypass -File .\scripts\start-local-test-stack.ps1 -EngineRoot "..\..\UnrealEngine"
```

자동 탐색도 지원:
- `C:\UnrealEngine`, `D:\UnrealEngine`
- `C:\Program Files\Epic Games\UE_*` 계열

Force 빌드 후 로컬 스택 실행(로비 + Dedicated):

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start-local-test-stack.ps1 -ForceRebuildStaged
```

Force 빌드 없이 실행(기존 Staged 서버 재사용, 없으면 자동 빌드):

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start-local-test-stack.ps1
```

Dedicated 서버만 실행(로비 서버는 이미 따로 켠 상태):

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start-local-test-stack.ps1 -SkipLobby
```

로컬 테스트 프로세스 종료:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\stop-local-test-stack.ps1
```

사설 서버(Server.exe) 빌드 후 실행:

```powershell
# 빌드
cmake -S ..\Server -B ..\Server\build
cmake --build ..\Server\build --config Release

# 실행
..\Server\build\Server.exe
```

참고:
- `start-local-test-stack.ps1`는 로비 서버 실행 파일을 상대경로로 탐색합니다.
  - `..\Server\build\Server.exe` (우선)
  - `..\Server\GameServer\build\Release\Server.exe`
  - `..\Server\GameServer\build\Server.exe`
- Dedicated 기본 맵은 `-Map` 미지정 시 `/Game/PersonalWorkSpace/wjtmd28/Loading_Level` 입니다.
- 실제 룸 인게임 템플릿 맵은 서버 런타임에서 `/Game/PersonalWorkSpace/wjtmd28/MyMap`을 스트리밍합니다.
